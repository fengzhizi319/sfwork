## 使用建议

1. **先读总览**：从 `00-项目总览/` 的白皮书和 PPT 建立整体认知。
2. **再按模块深入**：根据角色进入 `02-前端开发/`、`03-后端开发/` 或 `04-Kuscia/`。
3. **交叉参考**：架构设计、部署运维、问题排查通常需要跨模块阅读。
4. **注意时效**：部分文档可能随版本迭代，以源码仓库最新文件为准。
