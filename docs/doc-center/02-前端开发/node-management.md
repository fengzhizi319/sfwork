# 页面需求：节点管理

> 包含 Center 节点注册（`/home?tab=node-management`、`/nodes`）与我的节点（`/my-node`）。

## 一、Center 节点注册（中文）

### 页面目标
让 CENTER 管理员集中管理已注册的 Kuscia 节点，包括新增、编辑地址、刷新状态、删除。

### 用户故事
- 作为管理员，我希望看到本 Center 下所有节点及其状态。
- 作为管理员，我希望新增一个节点，输入节点 ID、名称、通讯地址、节点类型。
- 作为管理员，我希望刷新节点状态，确认节点与 Kuscia 的连通性。

### 主界面元素
- 搜索框：按节点名称/ID 过滤。
- 计算模式过滤：MPC / TEE / MPC&TEE。
- 节点列表/卡片：节点 ID、名称、通讯地址、节点类型、状态。
- 操作按钮：新增节点、刷新状态、编辑、删除。

### 操作与反馈

| 操作 | 反馈 |
|---|---|
| 新增节点 | 弹出表单抽屉，填写后提交；成功后列表刷新 |
| 编辑通讯地址 | 弹出编辑抽屉，保存后状态重新检测 |
| 刷新状态 | 调用 `/node/refresh`，列表状态更新 |
| 删除节点 | 二次确认，删除后不可恢复 |

### 业务规则
- 节点类型包括 MPC、TEE、MPC&TEE。
- 内置节点（alice/bob/tee）不可删除或编辑。
- 删除前需校验该节点无运行中任务或未释放资源。

## 二、英文节点管理（/nodes）

### 页面目标
新增的中心节点管理页，字段更轻量，面向国际化用户。

### 主界面元素
- 列表字段：Node ID、Name、Address、Type、Status、Description。
- 顶部操作：Add、Refresh、搜索。
- 行级操作：Edit、Delete。

### 新增/编辑节点表单

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| Name | Input | 是 | 节点显示名称 |
| Address | Input | 是 | gRPC/HTTP 通讯地址 |
| Node Type | Select | 是 | MPC / TEE / TRUSTED |
| Description | TextArea | 否 | 节点用途描述 |
| Certificate | TextArea/File | 是（部分类型） | 节点证书 |
| Private Key | TextArea/File | 是（部分类型） | 节点私钥 |

## 三、我的节点（/my-node）

### 页面目标
让机构管理员查看并维护当前机构所属节点的基本信息、实例状态和认证信息。

### 用户故事
- 作为机构管理员，我希望查看当前节点的 ID、地址、协议、证书配置状态。
- 作为机构管理员，我希望复制节点的公钥、认证码、token。
- 作为 AUTONOMY 用户，我希望在多个节点间切换。

### 主界面元素
- 节点基础信息卡片：节点 ID、通讯地址（可编辑）、通讯协议、证书配置状态、公钥、认证码、token。
- 节点实例列表：HostName、状态、系统版本、创建时间、最后心跳时间。
- 操作按钮：编辑通讯地址、重新生成 token、切换节点（AUTONOMY）。

### 操作与反馈

| 操作 | 反馈 |
|---|---|
| 编辑通讯地址 | 保存后调用 `/node/update`，成功后刷新 |
| 复制公钥/token | 点击复制，Toast 提示 |
| 生成新 token | 二次确认，成功后显示新 token |
| 切换节点 | 下拉选择后重载页面数据 |

### 业务规则
- AUTONOMY 模式支持节点切换；EDGE/CENTER 模式通常固定。
- 节点认证信息属于敏感内容，需支持一键复制但不宜明文长期展示。
