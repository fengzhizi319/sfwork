# 页面需求：模型管理与结果管理

## 一、模型管理

### 1.1 模型列表（/dag 顶部“模型管理”）

#### 页面目标
展示当前项目下训练得到的模型包，支持发布、下线、废弃、删除等全生命周期管理。

#### 主界面元素
- 模型列表：模型名称、ID、描述、发布状态、提交时间、操作。
- 状态标签：
  - 待发布
  - 已发布
  - 已下线
  - 已废弃
  - 发布失败
  - 发布中
- 操作按钮：发布、废弃、下线、删除、查看服务。

#### 操作与反馈

| 操作 | 反馈 |
|---|---|
| 发布 | 调用模型服务创建接口，状态变为“发布中”，成功后“已发布” |
| 下线 | 状态变为“已下线”，服务不可用 |
| 废弃 | 状态变为“已废弃”，不可再发布 |
| 删除 | 二次确认，删除后从列表移除 |
| 查看服务 | 打开服务详情抽屉，展示访问地址、资源配置 |

#### 业务规则
- P2P 模式下仅 owner 可操作。
- 已归档项目不可发布模型。
- 发布中的模型不可重复发布。

### 1.2 模型提交（/model-submission）

#### 页面目标
从 DAG 画布中选择一个或多个模型组件，打包提交到模型管理。

#### 主界面元素
- 可提交模型组件列表。
- 提交进度 Drawer。
- 取消/退出提示。

#### 业务规则
- 退出提交页面前提示“退出将中断提交进程”。
- 提交成功后模型出现在模型管理列表。

### 1.3 模型服务详情

#### 主界面元素
- 服务名称、状态、访问地址。
- 资源配置：内存、CPU、模型路径。
- 特征配置与样本表信息。
- 操作：删除服务。

## 二、结果管理

### 2.1 结果列表（/edge?tab=result、/node?tab=result）

#### 页面目标
让数据提供方或建模工程师查看节点上生成的各类结果产物，支持下载与重新获取。

#### 主界面元素
- 搜索：按结果 ID / 来源项目 / 训练流。
- 类型过滤：规则 / 模型 / 表 / 报告。
- 结果表列表：结果 ID、类型、来源项目、所属训练流、所属节点、生成时间、状态。
- 操作：下载、重新获取、查看详情。

### 2.2 结果详情 Drawer

#### 主界面元素
- 结果元数据：名称、类型、生成时间、状态。
- DAG 预览：展示产生该结果的训练流快照。
- 下载按钮（部分数据源类型不支持）。

### 2.3 操作与反馈

| 操作 | 反馈 |
|---|---|
| 下载 | 浏览器触发下载，失败时提示 |
| 重新获取 | 从 Kuscia 重新拉取结果元数据 |
| 查看详情 | 打开结果详情 Drawer |

### 2.4 业务规则
- 下载受数据权限与数据源类型限制（如 ODPS 可能不支持直接下载）。
- TEE 结果下载需先完成 TEE 下载审批。
- 结果按来源项目和所属节点隔离展示。
