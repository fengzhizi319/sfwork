# 业务系统集成指南

> 本文档说明业务系统如何将隐私计算能力（脱敏、差分隐私、K-匿名、查询混淆、分类分级）集成到自身架构中。
>
> 来源：`docs/隐私计算平台-团队汇报与落地规划.md` 第十一章

---

## 1. 集成模式总览

平台提供两种集成模式：**嵌入式 SDK**（Java / Go）和 **Sidecar Agent**（Python REST + gRPC），业务方可根据技术栈、部署约束和性能要求灵活选择。

```
┌──────────────────────────────────────────────────────────────────┐
│                        业务应用系统                               │
│  (HIS / EMR / CRM / 数据中台 / 自研后端)                         │
│                                                                  │
│  ┌─────────────────────┐     ┌──────────────────────────────┐    │
│  │  模式 A：嵌入式 SDK  │     │  模式 B：Sidecar Agent        │    │
│  │  Java SDK / Go SDK  │     │  REST :8079  gRPC :50051     │    │
│  │  函数调用，零网络开销  │     │  HTTP/gRPC 调用，跨语言统一   │    │
│  └─────────┬───────────┘     └──────────┬───────────────────┘    │
│            │                            │                        │
│            ▼                            ▼                        │
│  ┌─────────────────────────────────────────────────────────┐     │
│  │              隐私处理能力层                               │     │
│  │  脱敏 │ 差分隐私 │ K-匿名 │ 查询混淆 │ 分类分级          │     │
│  └─────────────────────────────────────────────────────────┘     │
└──────────────────────────────────────────────────────────────────┘
```

| 维度 | 模式 A：嵌入式 SDK | 模式 B：Sidecar Agent |
|------|-------------------|----------------------|
| **适用语言** | Java 17+ / Go 1.21+ | 任意语言（REST/gRPC） |
| **部署形态** | Maven/Go Module 依赖，进程内调用 | 独立容器/进程，网络调用 |
| **延迟** | 微秒级（函数调用） | 毫秒级（HTTP/gRPC） |
| **运维复杂度** | 低（随应用部署） | 中（需管理 Sidecar 生命周期） |
| **升级方式** | 升级依赖版本，重新发布应用 | 升级 Agent 镜像/包，应用无感 |
| **推荐场景** | 高性能、同语言后端、数据不出进程 | 多语言混合栈、Python/.NET/Node.js 后端、需独立升级 |

## 2. 模式 A：嵌入式 SDK 集成

### 2.1 Java SDK 集成（适用于 SecretPad 后端、Spring Boot 业务系统）

**步骤 1：添加 Maven 依赖**

```xml
<dependency>
  <groupId>com.github.fengzhizi319</groupId>
  <artifactId>privacy-java-sdk</artifactId>
  <version>0.1.0-SNAPSHOT</version>
</dependency>
```

**步骤 2：在 Service 层注入调用**

```java
import com.github.fengzhizi319.privacy.sdk.PrivacyClient;
import com.github.fengzhizi319.privacy.sdk.classification.ClassificationClient;

@Service
public class PatientDataService {

    private final PrivacyClient privacyClient = new PrivacyClient();
    private final ClassificationClient classClient = new ClassificationClient();

    /**
     * 查询患者信息时自动脱敏敏感字段
     */
    public PatientDTO queryPatient(String patientId, String callerRole) {
        PatientDTO patient = patientRepo.findById(patientId);

        // 根据调用方角色自动脱敏
        patient.setMobile(
            privacyClient.masking().maskValue("mobile", patient.getMobile(), callerRole)
        );
        patient.setIdCard(
            privacyClient.masking().maskValue("id_card", patient.getIdCard(), callerRole)
        );
        return patient;
    }

    /**
     * 数据导出前批量分类分级 + 隐私处理
     */
    public ExportResult prepareExport(List<Map<String, Object>> rows) {
        // 1. 分类分级：识别表中哪些字段敏感
        var tableResult = classClient.classification()
            .classifyTable(rows.get(0).keySet(), rows, null);

        // 2. 对 L3 以上字段自动脱敏
        for (var field : tableResult.getFieldResults()) {
            if (field.getFinalLevel().ordinal() >= 3) {
                rows.forEach(row -> {
                    String val = String.valueOf(row.get(field.getFieldName()));
                    row.put(field.getFieldName(),
                        privacyClient.masking().maskValue(
                            field.getFieldName(), val, "data_export"));
                });
            }
        }
        return new ExportResult(rows, tableResult);
    }
}
```

**步骤 3：与 SecretPad 后端集成**

SecretPad 后端本身是 Spring Boot 应用，可直接引入 `privacy-java-sdk`：

```java
// 在 secretpad-service 模块的 pom.xml 中添加依赖
// 在数据查询/导出 Controller 中调用分类分级 + 脱敏
@RestController
@RequestMapping("/api/v1alpha1/datatable")
public class DataTableController {

    @Autowired
    private DataTableService dataTableService;

    private final PrivacyClient privacyClient = new PrivacyClient();

    @GetMapping("/{id}/preview")
    public Result<TablePreviewVO> preview(@PathVariable String id) {
        TablePreviewVO data = dataTableService.preview(id);
        // 预览时对敏感字段自动脱敏
        data.getRows().forEach(row -> {
            row.forEach((field, val) -> {
                var cls = classClient.classification()
                    .classifyField(field, String.valueOf(val), null);
                if (cls.getFinalLevel().ordinal() >= 3) {
                    row.put(field, privacyClient.masking()
                        .maskValue(field, String.valueOf(val), "preview"));
                }
            });
        });
        return Result.success(data);
    }
}
```

### 2.2 Go SDK 集成（适用于 Kuscia 插件、Go 微服务）

**步骤 1：添加 Go Module 依赖**

```bash
go get github.com/fengzhizi319/privacy-go-sdk
```

**步骤 2：在业务逻辑中调用**

```go
package service

import (
    "github.com/fengzhizi319/privacy-go-sdk/sdk"
)

type ReportService struct {
    privacyClient *sdk.Client
    clsClient     *sdk.ClassificationClient
}

func NewReportService() *ReportService {
    return &ReportService{
        privacyClient: sdk.NewClient(),
        clsClient:     sdk.NewClassificationClient(),
    }
}

// 报表数据返回前自动脱敏
func (s *ReportService) GetReport(queryCtx QueryContext) (*Report, error) {
    rows, err := s.queryDB(queryCtx)
    if err != nil {
        return nil, err
    }

    // 对每条记录的敏感字段脱敏
    for _, row := range rows {
        if mobile, ok := row["mobile"].(string); ok {
            row["mobile"] = s.privacyClient.MaskValue("mobile", mobile, queryCtx.CallerRole)
        }
        if idCard, ok := row["id_card"].(string); ok {
            row["id_card"] = s.privacyClient.MaskValue("id_card", idCard, queryCtx.CallerRole)
        }
    }
    return &Report{Rows: rows}, nil
}

// 数据导出前批量分类 + K-匿名处理
func (s *ReportService) ExportAnonymized(schema []string, rows []map[string]any) ([]map[string]any, error) {
    // 1. 分类分级
    tableResult := s.clsClient.ClassifyTable(schema, rows, nil)

    // 2. 对 L3+ 字段做 K-匿名泛化
    hierarchies := map[string]sdk.GeneralizationHierarchy{
        "age":     sdk.AgeHierarchy(),
        "zipcode": sdk.ZipcodeHierarchy(),
    }
    qiFields := []string{} // 准标识符字段
    for _, f := range tableResult.FieldResults {
        if f.FinalLevel >= sdk.Level3 {
            qiFields = append(qiFields, f.FieldName)
        }
    }

    var result []map[string]any
    for _, row := range rows {
        anon := s.privacyClient.KAnonymizeRecord(row, qiFields, hierarchies, 5)
        result = append(result, anon)
    }
    return result, nil
}
```

## 3. 模式 B：Sidecar Agent 集成

Privacy Local Agent 以独立进程方式运行，业务系统通过 REST 或 gRPC 调用，**适合 Python / .NET / Node.js / PHP 等非 Java/Go 技术栈**，或需要隐私能力独立升级的场景。

### 3.1 部署 Agent

```bash
# Docker 部署（推荐）
docker run -d --name privacy-agent \
  -p 8079:8079 -p 50051:50051 \
  privacy-local-agent:0.1.0

# 或 K8s 部署（Helm）
helm install privacy-agent ./charts/privacy-local-agent \
  --set replicaCount=2 \
  --set resources.limits.memory=512Mi
```

### 3.2 Python 业务系统集成（REST）

```python
import requests

class PrivacyHelper:
    """封装 Privacy Agent 调用，集成到 Python 业务系统"""

    def __init__(self, agent_url="http://privacy-agent:8079"):
        self.agent_url = agent_url

    def mask_if_sensitive(self, field_name: str, value: str, context: str = "default") -> str:
        """先分类，若敏感则脱敏"""
        # 分类分级
        cls_resp = requests.post(f"{self.agent_url}/v1/privacy/classify/field", json={
            "field_name": field_name, "value": value, "params": {}
        }).json()

        level = cls_resp.get("finalLevel", "L1")
        if level >= "L3":
            # 脱敏
            mask_resp = requests.post(f"{self.agent_url}/v1/privacy/mask", json={
                "field_name": field_name, "value": value, "context": context
            })
            return mask_resp.json().get("result", value)
        return value

    def export_with_dp(self, data: list[dict], epsilon: float = 1.0) -> list[dict]:
        """导出时使用差分隐私保护数值字段"""
        result = []
        for row in data:
            noisy_row = {}
            for k, v in row.items():
                if isinstance(v, (int, float)):
                    dp_resp = requests.post(f"{self.agent_url}/v1/privacy/dp/count", json={
                        "values": [float(v)], "epsilon": epsilon, "mechanism": "laplace"
                    })
                    noisy_row[k] = dp_resp.json().get("result", v)
                else:
                    noisy_row[k] = v
            result.append(noisy_row)
        return result

# 在 Flask / FastAPI / Django 中使用
from fastapi import FastAPI
app = FastAPI()
privacy = PrivacyHelper()

@app.get("/api/patients/{patient_id}")
async def get_patient(patient_id: str):
    patient = await db.get_patient(patient_id)
    # 返回前自动脱敏
    patient["mobile"] = privacy.mask_if_sensitive("mobile", patient["mobile"], "api_query")
    patient["id_card"] = privacy.mask_if_sensitive("id_card", patient["id_card"], "api_query")
    return patient
```

### 3.3 Node.js / .NET 业务系统集成（gRPC）

Agent 提供 gRPC 端点（端口 50051），任何支持 gRPC 的语言均可直接调用：

```javascript
// Node.js 示例
const grpc = require('@grpc/grpc-js');
const protoLoader = require('@grpc/proto-loader');

const PROTO_PATH = 'proto/privacy.proto';
const packageDef = protoLoader.loadSync(PROTO_PATH, {
    keepCase: true, longs: String, enums: String, defaults: true, oneofs: true
});
const proto = grpc.loadPackageDefinition(packageDef);

const client = new proto.privacy.PrivacyService(
    'privacy-agent:50051', grpc.credentials.createInsecure()
);

// 脱敏调用
client.Mask({
    field_name: 'mobile',
    value: '13812345678',
    context: 'crm_query'
}, (err, response) => {
    console.log('脱敏结果:', response.result);
});
```

### 3.4 与现有数据管道集成

对于 ETL / 数据导出 / 报表生成等批量场景，Agent 可作为管道中的一个处理节点：

```python
# 数据导出管道示例
import pandas as pd
import requests

AGENT_URL = "http://privacy-agent:8079"

def privacy_aware_export(df: pd.DataFrame, output_path: str):
    """带隐私保护的数据导出管道"""
    # Step 1: 表级分类分级
    schema = list(df.columns)
    rows = df.head(100).to_dict('records')
    cls_resp = requests.post(f"{AGENT_URL}/v1/privacy/classify/table", json={
        "schema": schema, "rows": rows, "params": {}
    }).json()

    # Step 2: 根据分类结果选择处理策略
    for field_info in cls_resp.get("field_results", []):
        fname = field_info["field_name"]
        level = field_info["final_level"]

        if level == "L1":
            continue  # 公开数据，不处理
        elif level == "L2":
            pass      # 低风险，记录日志即可
        elif level == "L3":
            # 中风险：脱敏
            df[fname] = df[fname].apply(
                lambda v: requests.post(f"{AGENT_URL}/v1/privacy/mask", json={
                    "field_name": fname, "value": str(v), "context": "export"
                }).json().get("result", v)
            )
        elif level == "L4":
            # 高风险：K-匿名泛化
            df = df.apply(lambda row: requests.post(
                f"{AGENT_URL}/v1/privacy/k_anonymize/record", json={
                    "record": row.to_dict(),
                    "quasi_identifiers": [fname],
                    "k": 5
                }).json().get("result", row.to_dict()), axis=1, result_type='expand')
        elif level == "L5":
            # 极高风险：直接移除
            df = df.drop(columns=[fname])

    # Step 3: 输出
    df.to_csv(output_path, index=False)
```

## 4. 典型业务集成场景

### 场景 1：医院 HIS 系统（Java）

```
HIS 后端 (Spring Boot)
  └── privacy-java-sdk（Maven 依赖）
        ├── 患者查询接口 → 自动脱敏手机号、身份证
        ├── 病历导出 → 分类分级 + K-匿名
        └── 统计报表 → 差分隐私计数/均值
```

- **集成方式**：嵌入式 SDK，函数调用零延迟
- **关键点**：在 Service 层拦截返回数据，根据字段分类结果自动选择脱敏策略
- **性能影响**：< 1ms/字段，对业务接口几乎无感

### 场景 2：数据中台（Python + Go 混合）

```
数据中台
  ├── 数据采集服务 (Go)  → privacy-go-sdk → 入库前分类标注
  ├── 数据治理服务 (Python) → Privacy Agent REST → 批量脱敏/匿名
  └── 数据服务 API (Go)  → privacy-go-sdk → 返回前实时脱敏
```

- **集成方式**：Go 服务用 SDK，Python 服务用 Agent Sidecar
- **关键点**：统一使用相同的分类分级规则，确保全链路策略一致
- **部署**：Agent 以 DaemonSet 形式部署在每个节点，Go SDK 随应用发布

### 场景 3：CRM / 客服系统（.NET / Node.js）

```
CRM 前端 (React)
  └── CRM 后端 (.NET / Node.js)
        └── Privacy Agent Sidecar (REST/gRPC)
              ├── 客户信息查询 → 脱敏展示
              ├── 工单导出 → K-匿名 + 差分隐私
              └── 营销查询 → 查询混淆
```

- **集成方式**：Agent Sidecar，通过 REST/gRPC 调用
- **关键点**：Agent 与业务服务同机部署或 K8s 同 Pod，降低网络延迟
- **升级**：隐私策略升级只需更新 Agent 镜像，业务代码无需改动

### 场景 4：SecretPad 平台自身集成

```
SecretPad 前端 (React)
  └── SecretPad 后端 (Spring Boot + privacy-java-sdk)
        ├── 数据表预览 → 实时分类 + 脱敏
        ├── 数据导出任务 → 分类 → 按级别选择处理原语
        ├── DAG 编排 → 隐私处理组件节点
        └── 审计日志 → 记录每次隐私处理操作
```

- **集成方式**：Java SDK 嵌入 SecretPad 后端
- **关键点**：分类分级结果写入 `DataTableFieldDO` 元数据，前端展示分类标签
- **审计**：每次脱敏/匿名操作记录到 `PrivacyAuditLogDO`，含操作人、时间、字段、策略

## 5. 集成最佳实践

| 实践 | 说明 |
|------|------|
| **先分类后处理** | 始终先调用分类分级确定敏感等级，再根据等级选择对应处理原语 |
| **策略与代码分离** | 通过 Profile YAML 配置隐私策略，避免硬编码规则到业务逻辑中 |
| **SDK 优先，Agent 兜底** | 同语言优先用 SDK（零延迟），跨语言或需独立升级时用 Agent |
| **批量处理用管道** | 大量数据导出场景，使用批量 API（`mask_record`、`k_anonymize_record`、`classify_table`）减少调用次数 |
| **隐私预算管控** | 差分隐私调用需跟踪预算消耗，避免超出项目配置的总预算 |
| **审计日志必开** | 所有隐私处理操作应记录审计日志，满足合规要求 |
| **灰度发布** | 隐私策略变更建议先在非生产环境验证分类准确率，再灰度上线 |

## 6. 集成架构对比

| 架构要素 | SDK 模式 | Agent 模式 |
|---------|---------|------------|
| **故障隔离** | SDK 异常影响宿主进程 | Agent 异常不影响业务进程 |
| **资源隔离** | 共享宿主进程 CPU/内存 | 独立资源限制（K8s limits） |
| **灰度升级** | 需重新发布应用 | 独立滚动升级 Agent |
| **监控** | 应用 APM 统一采集 | Agent `/metrics` + 应用侧埋点 |
| **安全边界** | 与应用同信任域 | 可独立配置 TLS + API Key 认证 |
| **多实例** | 每个应用实例内置一份 | 可共享 Agent 实例（网关负载均衡） |
