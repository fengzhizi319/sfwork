# 本地多模态大模型分类分级网关 PRD

## 1. 业务背景与痛点
根据 [分类分级算法设计](file:///home/charles/code/sfwork/docs/algorithm/%E5%88%86%E7%B1%BB%E5%88%86%E7%BA%A7%E7%AE%97%E6%B3%95%E8%AE%BE%E8%AE%A1.md) 的规划，在第三层检测逻辑中，我们需要处理高度非结构化的医疗数据，包括：
1. **病例图片**（例如 CT 报告截图、化验单扫描件、影像诊断书）。
2. **手写病例图片**（医生手写的门诊病历、处方单，字迹潦草且排版复杂）。
3. **极非结构化文本**（OCR 识别出的凌乱文本、跨段落的隐式病症关联）。

传统的规则引擎（RE2/FST）和轻量级命名实体识别（Small-NER）模型对此类复杂的多模态与非结构化输入无能为力。为了实现高召回、低漏报的自动定级，必须引入**本地私有化部署的多模态大模型（Vision-Language Model, VLM）**作为第三层语义判别器。

## 2. 产品定位与目标
- **定位**：本地安全域内运行的多模态推理服务，支持接收图片或文本，自动提取其中的敏感医疗信息并输出分类分级标签。
- **目标**：
  - **医学 OCR 与 HTR（手写体识别）**：能够高精度识别中文手写病例和医疗影像报告中的文本。
  - **语义定级推理**：基于识别出的文本和上下文，结合预置的 L1~L5 分类矩阵进行零样本（Zero-shot）逻辑推理，输出精准的定级标签。
  - **绝对隐私安全**：模型必须 100% 本地化执行，不得向外部公网接口发送任何敏感的病例数据。
  - **轻量化适配**：能够在本地的开发环境（GeForce RTX 5060 Laptop GPU，8GB 显存）中流畅运行。

## 3. 大模型选型 (Model Selection)

针对本地部署和 8GB 显存的硬件限制，我们选择 **Qwen2-VL-2B-Instruct** 作为本地多模态大模型：

| 维度 | Qwen2-VL-2B-Instruct | 选型优势说明 |
| --- | --- | --- |
| **参数量** | 2.2 Billion | 规模适中，在 FP16 推理下仅需约 5GB 显存，可在 8GB RTX 5060 显卡上流畅运行，留有足够的上下文缓存空间。 |
| **多模态能力** | 视觉-语言双模态 | 原生支持图像输入，无需外接额外的 OCR 引擎，减少了多组件流水线带来的误差传递。 |
| **中文及手写体识别** | 顶尖的中文文档理解 | 在中文 OCR、手写体识别 (HTR) 以及医疗报告等长文本图表理解任务上处于开源社区领先水平，极其擅长识别潦草的手写病历。 |
| **指令遵循与格式化输出** | 优秀的 JSON Schema 遵循 | 支持通过 Prompt 强约束输出标准的 JSON 格式，便于系统自动解析分类分级结果。 |

## 4. 功能性需求 (Functional Requirements)

### 4.1 多模态输入自适应识别 (Input Auto-Detection)
分类服务接口 `classify_field` 在接收输入值 `value` 时，大模型分类器必须能够自动识别并处理以下三种输入格式：
1. **本地图片路径**：若 `value` 为字符串且是本地存在的图片文件路径（如 `/path/to/report.jpg`），自动读取并将其作为图像输入大模型。
2. **Base64 编码图片**：若 `value` 为 `data:image/...;base64,...` 格式的 DataURI 或符合 base64 规范的图像字节编码，网关自动解码后送入大模型。
3. **文本内容**：若不满足上述条件，直接将其作为纯文本文档送入大模型进行语义分类。

### 4.2 零样本语义定级与结构化输出 (Zero-shot Inference)
大模型在接收图像或文本输入后，需结合系统 Prompt（包含 L1~L5 分类矩阵定义与定级要求），输出如下标准的 JSON 字典：
```json
{
  "final_level": "L4",
  "sub_category": "MEDICAL_SENSITIVE_DISEASE",
  "confidence": 0.92,
  "reasoning": "图片中包含‘抗逆转录病毒治疗’等字样，推断属于 HIV 敏感病史，评定为 L4 级高风险数据",
  "needs_human_review": false
}
```
禁止输出任何自由文本，确保系统能够 100% 解析成功。

### 4.3 模型下载与环境准备工具 (Tooling)
- 提供一键下载脚本，支持从国内高速镜像（如 ModelScope）或 Hugging Face 下载 `Qwen2-VL-2B-Instruct` 权重。
- 支持断点续传和自动保存到指定的本地模型目录。

### 4.4 优雅降级与防御性设计 (Graceful Fallback)
- **环境检测**：大模型初始化时应检测 CUDA 设备。若本地显卡不满足要求，或者模型文件未下载，应输出清晰的告警，并自动降级回规则引擎与轻量级 NER 引擎（No-Op LLM Fallback），保证系统核心数据流不会中断。

## 5. 非功能性需求 (Non-Functional Requirements)
- **显存与硬件控制**：
  - 大模型运行时显存占用应控制在 5.5GB 以下。
- **推理延迟**：
  - 纯文本推理时间限制在 500ms 以内。
  - 图片/手写病例识别与定级时间控制在 2.5 秒以内。
