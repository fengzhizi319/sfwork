# 数据分类分级功能总体设计

## 1. 背景与目标

在 sfwork 二次开发环境中，为 SecretPad 训练流新增一套**数据分类分级**能力：

- 用户新建项目时可以选择「数据分类分级」训练流模板。
- 模板自动构造 `read_data/datatable → privacy/data_classification` 的 DAG。
- `privacy/data_classification` 组件基于 `privacy-local-agent` 的分类规则引擎实现，支持**自动模式**与**高级模式**两种参数配置方式。
- 组件输出分类后的数据表（附加等级列）和一份列级汇总报告。
- 全流程最终通过 `scripts/dev-start.sh` 跑通。

## 2. 端到端架构

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                              SecretPad Frontend                              │
│  新建项目 → 选择「数据分类分级」模板 → DAG 画布 → 配置参数（自动/高级）        │
│         → 点击运行                                                           │
└───────────────────────────────┬─────────────────────────────────────────────┘
                                │ REST /api/v1alpha1/graph/start
┌───────────────────────────────▼─────────────────────────────────────────────┐
│                              SecretPad Backend                               │
│  GraphServiceImpl.startGraph → ProjectJob → JobChain                         │
│    → KusciaJobConverter → CreateJobRequest(sfNodeEvalParam.comp_id)          │
└───────────────────────────────┬─────────────────────────────────────────────┘
                                │ gRPC
┌───────────────────────────────▼─────────────────────────────────────────────┐
│                                   Kuscia                                     │
│  创建 KusciaJob/KusciaTask → 调度 Pod → 挂载 task-config.conf                │
└───────────────────────────────┬─────────────────────────────────────────────┘
                                │ container
┌───────────────────────────────▼─────────────────────────────────────────────┐
│                              SecretFlow Container                            │
│  secretflow.kuscia.entry → preprocess_sf_node_eval_param → comp_eval         │

│    → DataClassification.evaluate(ctx)                                        │

└─────────────────────────────────────────────────────────────────────────────┘
```

## 3. 模块划分

| 模块 | 位置 | 职责 |
|------|------|------|
| 算法层 | `secretflow/privacy/data_classification/` | 从 `privacy-local-agent` 移植规则引擎与模型，提供 `classify_dataframe` |
| 组件层 | `secretflow/secretflow/component/privacy/data_classification.py` | SecretFlow 组件封装：`@register`、`Field`、`evaluate` |
| 后端元数据 | `secretpad/config/components/secretflow.json` | 组件定义（domain/name/version/attrs/inputs/outputs） |
| 后端 i18n | `secretpad/config/i18n/secretflow.json` | 组件与参数中文翻译 |
| 前端模板 | `secretpad/frontend-src/.../pipeline/templates/pipeline-template-data-classification.ts` | 训练流模板内容 |
| 前端注册 | `secretpad/frontend-src/.../pipeline/index.ts` | 模板注册到 `PipelineTemplatesRegistry` |

## 4. 组件设计

### 4.1 基本元数据

- **domain**: `privacy`
- **name**: `data_classification`
- **version**: `1.0.0`
- **comp_id**: `privacy/data_classification:1.0.0`
- **输入**: `sf.table.individual`
- **输出**:
  - `output_ds`: `sf.table.individual` —— 原表 + `__final_level__`、`__needs_review__`、`__tags_json__` 列
  - `report`: `sf.report` —— 列级汇总统计

### 4.2 参数模式

使用 SecretFlow `AT_UNION_GROUP` 实现「自动/高级」模式切换。

#### 自动模式（auto）

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `mode` | union | `auto` | 模式选择 |
| `auto/default_level` | string | `L3` | 未命中规则时的默认敏感度等级 |

#### 高级模式（advanced）

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `mode` | union | `auto` | 模式选择 |
| `advanced/enable_rule_engine` | bool | `true` | 是否启用规则引擎 |
| `advanced/enable_small_ner` | bool | `false` | 是否启用 Small-NER（V1 为 no-op） |
| `advanced/enable_llm` | bool | `false` | 是否启用 LLM（V1 为 no-op） |
| `advanced/icd10_l4_intervals` | string(JSON) | `[{"start":"B20","end":"B24"},{"start":"F20","end":"F29"},{"start":"C00","end":"C97"}]` | 需要提升为 L4 的 ICD-10 区间 |
| `advanced/manual_override` | string(JSON) | `{}` | 字段名 → 强制等级覆盖 |

### 4.3 输出表 schema

| 列 | 类型 | 说明 |
|----|------|------|
| 原表所有列 | 原类型 | 保留 |
| `__final_level__` | string | 该记录最高敏感度等级 |
| `__needs_review__` | bool | 是否需要人工复核 |
| `__tags_json__` | string | 命中的标签 JSON 序列化 |

### 4.4 报告内容

| 列 | 类型 | 说明 |
|----|------|------|
| `column` | string | 字段名 |
| `final_level` | string | 该字段最高敏感度 |
| `hit_count` | int | 命中规则行数 |
| `categories` | string | 命中的规则类别 JSON |

## 5. 与 privacy-local-agent 的复用关系

| privacy-local-agent | SecretFlow 中的使用方式 |
|---------------------|------------------------|
| `classification_models.py` | 整体移植到 `secretflow/privacy/data_classification/models.py`，保留 pydantic v2 模型 |
| `classification.py` 的 `DefaultRuleEngine` | 移植到 `secretflow/privacy/data_classification/classification.py`，保留全部 L1 规则 |
| `ClassificationAPI.classify_table` / `classify_dataframe` | 移植并适配为纯本地调用，剥离 REST/gRPC 语义 |
| `profile.py` / `ParameterResolver` | 不移植；参数合并逻辑简化为组件 attr → `ClassificationParams` |
| REST/gRPC 服务层 | 完全不引入 |

## 6. 开发顺序

1. **阶段 0**：编写本文档（`01-总体设计.md`）。
2. **阶段 1**：编写 `02-SecretFlow组件设计.md`，然后实现 `secretflow/privacy/data_classification/` 与 `secretflow/secretflow/component/privacy/data_classification.py`。
3. **阶段 2**：编写 `03-SecretPad后端配置.md`，刷新 `secretpad/config/components/secretflow.json` 与 `secretpad/config/i18n/secretflow.json`。
4. **阶段 3**：编写 `04-前端模板与参数UI.md`，实现前端训练流模板并注册。
5. **阶段 4**：编写 `05-端到端测试计划.md`，执行 `scripts/dev-start.sh` 全流程验证。
6. **阶段 5**：运行中遇到的问题补充到 `06-问题与解决方案.md`。

## 7. 关键约定

- V1 仅实现规则引擎（L1_RULE），Small-NER 与 LLM 保留参数开关但实现为 no-op，以保证稳定可测。
- 组件仅支持 `sf.table.individual`，单方数据即可运行，无需跨多方 MPC。
- 所有 JSON 属性在前端以字符串形式传递，组件内部解析。
- 输出报告使用 SecretFlow `Reporter` 生成 `sf.report`。
