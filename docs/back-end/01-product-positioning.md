# 01 产品定位与边界

## 1.1 产品定位

SecretPad 后端是 SecretFlow 隐私计算平台的核心**业务中台**，承担以下职责：

- **统一接入层**：为 Web 前端、CLI、第三方系统提供 RESTful API。
- **业务编排层**：管理项目、节点、数据、训练流、作业、模型、审批等核心业务对象。
- **跨域协作层**：通过投票审批、消息通知、数据同步实现多机构间的可信协作。
- **Kuscia 适配层**：把业务对象转换为 Kuscia 的 Domain、DomainData、DomainRoute、Job 等概念，并监听其状态变化。

一句话定义：
> SecretPad 后端让多方隐私计算项目从“人拉肩扛”变为可编排、可审批、可审计的平台化协作。

## 1.2 目标用户

| 用户 | 需求 |
|---|---|
| 平台运营方 | 管理节点、监控系统、处理跨机构审批 |
| 数据提供方 | 注册数据、授权项目、保证数据不出域 |
| 算法工程师 | 通过 API 编排 DAG、启停作业、获取结果 |
| 机构管理员 | 管理机构节点、Token、证书、用户账号 |
| 审计合规人员 | 查看审批记录、操作日志、数据血缘 |

## 1.3 核心价值

1. **屏蔽底层复杂度**：前端无需关心 Kuscia gRPC、DomainDataGrant、Job 定义等细节。
2. **多方协作可信**：任何跨机构的资源变更都需要投票审批与数字签名。
3. **状态一致性**：CENTER/EDGE/P2P 多模式下的数据同步与任务状态回流。
4. **可扩展性**：新组件、新数据源类型、新部署模式可通过插件化方式接入。

## 1.4 产品边界

SecretPad 后端不直接负责：
- Kuscia 集群的部署与容器生命周期（由 Kuscia 自身负责）。
- SecretFlow 算法组件的实现（只管理组件元数据与参数）。
- 底层网络安全与 mTLS 证书管理（由 Kuscia 与基础设施负责，SecretPad 只调用接口）。
- 大规模离线数据分析执行（由 SecretFlow 在 Kuscia 调度的容器中执行）。

SecretPad 后端需要负责：
- 业务对象的持久化与一致性。
- 与 Kuscia 的 gRPC 调用适配。
- 多模式数据同步策略。
- 用户认证、接口鉴权、投票签名验证。
