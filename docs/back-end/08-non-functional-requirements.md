# 08 非功能性需求

## 8.1 性能

| 指标 | 目标 | 说明 |
|---|---|---|
| 登录接口 | P99 < 500ms | 含密码哈希与 Token 生成 |
| 项目列表 | P99 < 300ms | 常见分页 20 条 |
| DAG 详情 | P99 < 500ms | 含节点与边反序列化 |
| 任务状态同步 | 端到端 < 10s | Kuscia 事件到前端展示 |
| 文件上传 | 受网络和存储限制 | 大文件支持断点续传（后续） |

## 8.2 安全

- **传输安全**：生产环境使用 HTTPS，Kuscia 间使用 mTLS。
- **密码安全**：用户密码使用 SHA256 + 盐（或更强算法）存储，禁止明文。
- **Token 安全**：Token 过期时间 24 小时，支持滑动更新；登出立即失效。
- **接口安全**：外部端口禁止访问内部接口；接口级与数据级权限校验。
- **签名安全**：投票消息使用 RSA-SHA256 签名，证书链校验。
- **敏感信息**：节点私钥、证书、数据库密码不提交到代码仓库，生产环境使用 KMS/环境变量。

## 8.3 可观测性

- **日志**：统一日志格式，支持按请求 ID 串联全链路。
- **指标**：暴露 JVM、HTTP、Kuscia gRPC 调用关键指标。
- **健康检查**：提供 `/actuator/health` 或自定义健康接口。
- **告警**：节点掉线、任务失败、同步异常等需接入告警。

## 8.4 兼容性

- **JDK 17** 运行环境。
- **Spring Boot 3.3.x**。
- 支持 SQLite（开发/测试）与 MySQL（生产）。
- KusciaAPI 版本兼容：向后兼容至少一个 Kuscia 小版本。

## 8.5 高可用

- CENTER 模式支持多实例部署，需共享数据库与 Quartz JDBC JobStore。
- SSE 连接需支持负载均衡下的粘性会话或重连机制。
- Kuscia gRPC Channel 需支持断线重连与优雅关闭。

## 8.6 扩展性

- 新数据源类型通过实现 `DatasourceHandler` 接入。
- 新 SecretFlow 组件通过组件元数据自动渲染，无需修改后端代码。
- 新部署模式通过 `EnvService` 与权限包装器扩展。

## 8.7 可维护性

- 代码需通过 Spotless、Checkstyle、SpotBugs 静态检查。
- 核心业务规则需有单元测试覆盖。
- API 变更需同步更新前端与文档。
