# 06 全局交互规范

## 6.1 页面加载与状态反馈

| 场景 | 规范 |
|---|---|
| 初始加载 | 显示全局或局部 Skeleton，避免白屏。 |
| 数据请求中 | 表格显示 Spin，按钮显示 loading，禁用重复提交。 |
| 空状态 | 根据上下文显示插画 + 文案 + 引导操作，如“暂无项目，立即创建”。 |
| 请求失败 | Toast 或 Modal 提示错误信息，关键操作提供重试按钮。 |
| 请求成功 | Toast 提示成功，列表/详情自动刷新。 |

## 6.2 表单与提交

- 必填项以红色星号标注，提交前做前端校验。
- 长表单分步骤或分组展示，避免一屏内信息过载。
- 提交按钮在接口返回前禁用，防止重复提交。
- 表单提交失败时保留用户已填写内容，并定位到第一个错误字段。

## 6.3 弹窗与抽屉

- 创建/编辑类操作优先使用抽屉（Drawer），保持上下文。
- 危险操作（删除、下线、废弃）使用二次确认 Modal。
- 需要多步骤引导的（创建项目）使用分步抽屉或 Wizard。
- 全局性操作（登录过期、系统公告）使用 Modal。

## 6.4 权限缺省

- 无权限访问的页面直接拦截或 404。
- 无权限的操作按钮不渲染或禁用，并显示 Tooltip 说明原因。
- 数据不可见时显示“无权限查看”空态。

## 6.5 实时与轮询

- 任务状态、节点状态、消息未读数采用轮询或 SSE 实时更新。
- 轮询频率建议：任务状态 3~5 秒，消息数 30 秒，节点状态 60 秒。
- 页面不可见时暂停轮询，减少资源消耗。

## 6.6 错误处理

| 错误码 | 前端表现 |
|---|---|
| 401 / Token 失效 | 清除本地登录态，跳转登录页 |
| 403 | 提示“无权限”，必要时返回首页 |
| 404 | 展示 404 页面 |
| 500 / 服务异常 | Toast + 重试按钮 |
| 业务校验失败 | 展示后端返回的具体错误信息 |

## 6.7 快捷键与辅助功能

- DAG 画布支持常见快捷键：保存（Ctrl+S）、运行（Ctrl+Enter）、撤销（Ctrl+Z）、删除（Delete）。
- 左侧菜单支持折叠快捷键。
- 关键操作按钮需有键盘可达性（Tab 顺序）。

## 6.8 国际化

- 所有用户-facing 文案需接入 i18n。
- 组件库文档、错误提示、空状态文案均需支持多语言。
- 当前默认中文，英文页面（dashboard/nodes/data-source/data-table/graphs）作为国际化试点。

## 6.9 响应式

- 管理后台以桌面端为主，最小适配 1280px 宽度。
- DAG 画布需支持缩放以适应不同屏幕。
- 表格在窄屏下支持横向滚动或列收起。
