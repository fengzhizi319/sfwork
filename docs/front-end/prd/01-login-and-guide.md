# 01 登录与引导

## 1.1 页面：登录页（/login）

### 需求背景
提供统一的账号密码登录入口，根据账号平台类型与首次登录状态进行首页分流。

### 页面流程

```mermaid
flowchart TD
    A[访问 /login] --> B{已登录?}
    B -->|是| C[按平台类型跳转到对应首页]
    B -->|否| D[显示登录表单]
    D --> E[输入账号密码]
    E --> F[前端 SHA256 哈希密码]
    F --> G[调用 /api/login]
    G --> H{登录成功?}
    H -->|否| I[显示错误提示]
    I --> E
    H -->|是| J[缓存 Token 与用户信息]
    J --> K{platformType?}
    K -->|AUTONOMY| L[/edge?ownerId=...]
    K -->|EDGE| M[/node?ownerId=...]
    K -->|CENTER 首次| N[/guide]
    K -->|CENTER 非首次| O[/home]
    K -->|CENTER 下 EDGE 子账号| P[/home]
```

### 低保真原型

```textn+--------------------------------------------------+
|                                                  |
|              [Logo] SecretPad                    |
|                                                  |
|          隐私计算平台 · 可视化控制中心            |
|                                                  |
|  +------------------------------------------+    |
|  | 账号                                     |    |
|  | [________________________]              |    |
|  |                                          |    |
|  | 密码                                     |    |
|  | [________________________]              |    |
|  |                                          |    |
|  | [ ] 记住我                               |    |
|  |                                          |    |
|  | [      登 录      ]                      |    |
|  |                                          |    |
|  | 账号或密码错误，请重试。                  |    |
|  +------------------------------------------+    |
|                                                  |
+--------------------------------------------------+
```

### 元素说明

| 元素 | 类型 | 说明 |
|---|---|---|
| 账号 | Input | 必填，支持字母/数字/下划线，长度 4-32 |
| 密码 | Password Input | 必填，前端提交前做 SHA256 哈希 |
| 记住我 | Checkbox | 可选，记住账号（非 Token） |
| 登录按钮 | Primary Button | 点击后禁用，接口返回前 loading |
| 错误提示 | Alert | 登录失败时显示，不预先提示具体字段 |

### 字段规则

| 字段 | 校验规则 |
|---|---|
| 账号 | 必填，长度 4-32，仅允许字母、数字、下划线、横线 |
| 密码 | 必填，长度 8-20，需包含大小写字母与数字（后端校验） |

### 异常与边界

| 场景 | 处理 |
|---|---|
| 账号不存在 | 提示“账号或密码错误” |
| 密码错误 | 提示“账号或密码错误”，记录登录失败次数 |
| 账号被锁定 | 提示“账号已锁定，请 X 分钟后重试” |
| Token 过期 | 自动跳转登录页，保留当前路径用于登录后回跳 |

### 权限说明
- 无需登录即可访问。
- 已登录用户访问 `/login` 时按规则重定向到首页。

## 1.2 页面：新手引导（/guide）

### 需求背景
首次登录的 CENTER 管理员通过图文 + Tour 了解平台核心概念与操作流程。

### 页面流程

```mermaid
flowchart TD
    A[CENTER 首次登录] --> B[/guide]
    B --> C[概念介绍页]
    C --> D[开始 Tour]
    D --> E[步骤1: 注册节点]
    E --> F[步骤2: 创建项目]
    F --> G[步骤3: 授权数据]
    G --> H[步骤4: 编排训练流]
    H --> I[步骤5: 查看结果与模型]
    I --> J[完成引导，进入 /home]
```

### 低保真原型

```textn+--------------------------------------------------+
|  [Logo]                 [体验新手引导]  [头像 ▼]  |
+--------------------------------------------------+
|                                                  |
|     欢迎使用 SecretPad                           |
|     5 步开启隐私计算协作                         |
|                                                  |
|  +-------+  +-------+  +-------+  +-------+     |
|  | 1.注册|  | 2.创建|  | 3.授权|  | 4.编排|     |
|  |  节点 |  |  项目 |  |  数据 |  |  流  |     |
|  +-------+  +-------+  +-------+  +-------+     |
|                                                  |
|        [跳过引导]          [开始体验]            |
|                                                  |
+--------------------------------------------------+
```

### 元素说明

| 元素 | 类型 | 说明 |
|---|---|---|
| 步骤卡片 | Card | 5 张，点击可查看该步骤说明 |
| 开始体验 | Primary Button | 进入 Tour |
| 跳过引导 | Text Button | 直接进入 `/home` |
| 体验新手引导 | Header 入口 | 非首次也可重新进入 |

### 交互说明

| 操作 | 反馈 |
|---|---|
| 点击步骤卡片 | 展开该步骤图文说明 |
| 点击“开始体验” | 进入 Tour，高亮页面元素 |
| 点击“下一步” | 高亮下一个元素并更新说明 |
| 点击“跳过” | 直接跳转 `/home` |

### 业务规则
- 仅 CENTER 平台首次登录进入。
- 引导过程中支持随时跳过。
- Tour 锚点元素包括：项目列表、创建项目按钮、数据授权入口、DAG 画布、模型管理。

### 权限说明
- 需要登录且 `platformType === CENTER`。
- CENTER 下的 EDGE 子账号不进入引导。
