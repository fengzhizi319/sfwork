# 00 全局规范

## 0.1 设计原则

1. **效率优先**：管理后台以信息密度和操作为核心，避免过度装饰。
2. **状态显性化**：任务、节点、审批、模型等关键对象必须有明确状态标签。
3. **权限前置**：无权限的入口不展示或禁用，并给出原因提示。
4. **容错友好**：所有异步操作提供加载、成功、失败、重试反馈。

## 0.2 布局框架

### 全局布局

```textn+------------------------------------------------------------------+
|  [Logo] 平台名称                                    [🔔] [头像 ▼]  |
+----------+-------------------------------------------------------+
|          |                                                       |
|  侧边菜单 |                  主内容区（面包屑 + 内容）              |
|          |                                                       |
|  · 首页   |                                                       |
|  · 节点   |                                                       |
|  · 数据   |                                                       |
|  · 项目   |                                                       |
|  · 消息   |                                                       |
|          |                                                       |
+----------+-------------------------------------------------------+
```

### 布局规则

- 顶部 Header 固定高度 56px。
- 左侧菜单默认展开宽度 208px，支持折叠至 64px。
- 主内容区最小宽度 1024px，内容区左右 padding 24px。
- 卡片列表默认 3 列（≥1440px）或 2 列（<1440px）。
- 表格操作列固定右对齐，支持横向滚动。

## 0.4 字体与排版

- 字体栈：`PingFang SC`, `Microsoft YaHei`, `-apple-system`, `BlinkMacSystemFont`, sans-serif。
- 标题：`font-size: 20px`, `font-weight: 500`。
- 正文：`font-size: 14px`, `line-height: 1.5`。
- 辅助文本：`font-size: 12px`。
- 数字指标使用等宽字体 `SFMono-Regular, Consolas, monospace`。

## 0.5 通用组件规范

### 按钮

| 类型 | 用途 |
|---|---|
| Primary | 主操作：创建、保存、运行、确定 |
| Default | 次要操作：取消、返回 |
| Dashed | 添加项：添加字段、添加节点 |
| Text | 链接型操作：查看详情、编辑、复制 |
| Danger | 危险操作：删除、下线、废弃、拒绝 |

### 表单

- 标签右对齐或顶对齐，必填用红色 `*` 标注。
- 输入框默认宽度 320px，长文本使用 TextArea。
- JSON 配置使用 Monaco Editor 或 JSON 文本域。
- 表单提交按钮在接口返回前禁用。
- 校验失败时滚动到首个错误字段。

### 表格

- 默认分页 10/20/50。
- 操作列最多 3 个外露操作，超过 3 个收入“更多”下拉。
- 列表为空时显示自定义空态插画 + 引导按钮。
- 加载中显示 Skeleton 或 Spin。

### 抽屉（Drawer）

- 创建/编辑类操作统一使用抽屉，宽度默认 520px，复杂配置可 720px。
- 抽屉标题格式：`操作 + 对象名`，如“创建项目”。
- 抽屉底部固定操作栏：[`取消` `确定`]。

### 弹窗（Modal）

- 危险操作二次确认。
- 多步骤引导使用 Modal  Wizard。

## 0.6 状态标签规范

| 对象 | 状态 | 颜色 |
|---|---|---|
| 节点 | Healthy / UnHealthy | 绿 / 红 |
| 数据表 | Available / UnAvailable | 绿 / 红 |
| 任务 | 未运行 / 运行中 / 成功 / 失败 / 已停止 | 灰 / 蓝 / 绿 / 红 / 橙 |
| 项目 | 审核中 / 已通过 / 已归档 / 已拒绝 | 蓝 / 绿 / 灰 / 红 |
| 模型 | 待发布 / 发布中 / 已发布 / 已下线 / 已废弃 / 发布失败 | 灰 / 蓝 / 绿 / 橙 / 灰 / 红 |
| 消息 | 待处理 / 已同意 / 已拒绝 | 蓝 / 绿 / 红 |

## 0.7 权限缺省规范

| 场景 | 处理方式 |
|---|---|
| 无页面权限 | 403 页面或重定向到首页 |
| 无操作权限 | 按钮不渲染或 disabled + Tooltip |
| 数据不可见 | 显示“暂无权限查看”空态 |
| 功能受部署模式限制 | 隐藏入口或置灰提示 |

## 0.8 加载与反馈规范

| 场景 | 反馈 |
|---|---|
| 页面初始加载 | 全局 Skeleton |
| 表格数据加载 | 表格内 Spin，表头保留 |
| 表单提交 | 按钮 loading，禁用重复提交 |
| 操作成功 | Toast 提示，自动关闭 3s |
| 操作失败 | Toast / Modal 提示，保留表单内容 |
| 异步长任务 | 进度条或轮询状态 |

## 0.9 错误处理规范

| HTTP/业务状态 | 前端表现 |
|---|---|
| 401 Token 失效 | 清除登录态，跳转 `/login` |
| 403 无权限 | 提示“无权限”，返回安全页面 |
| 404 | 展示 404 页面 |
| 500/服务异常 | Toast + 重试按钮 |
| 业务校验失败 | 展示后端返回的具体 message |

## 0.10 实时更新规范

- 任务状态：3~5s 轮询或 SSE。
- 消息未读数：30s 轮询。
- 节点/路由状态：60s 轮询，支持手动刷新。
- 页面不可见时暂停轮询。
