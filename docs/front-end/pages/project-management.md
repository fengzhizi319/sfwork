# 页面需求：项目管理

> 包含 Center 项目列表、P2P 我的项目、创建项目。

## 一、Center 项目列表（/home?tab=project-management）

### 页面目标
让 CENTER 管理员以卡片方式浏览和管理所有项目，快速进入项目空间。

### 用户故事
- 作为管理员，我希望按计算模式筛选项目。
- 作为管理员，我希望看到每个项目的参与节点数、训练流数、任务数。
- 作为管理员，我希望删除废弃项目，但需要通过项目名称二次确认。

### 主界面元素
- 搜索框：按项目名称搜索。
- 计算模式过滤：全部 / 管道 / 枢纽。
- 项目卡片列表：
  - 项目名称
  - 计算模式标签
  - 参与节点数 / 训练流数 / 任务数
  - 创建时间
  - 操作：进入项目、编辑、删除
- 新手引导 Tour 锚点。

### 操作与反馈

| 操作 | 反馈 |
|---|---|
| 进入项目 | 跳转 `/dag?projectId=xxx` |
| 编辑 | 打开编辑抽屉，可修改名称/描述 |
| 删除 | 弹窗要求输入项目名称确认，删除后刷新列表 |
| 创建项目 | 打开创建项目抽屉 |

### 业务规则
- 删除项目前需校验项目下无画布/训练流或相关任务。
- CENTER 下的 EDGE 子账号只能看到被授权的项目。

---

## 二、P2P 我的项目（/edge?tab=my-project）

### 页面目标
让 AUTONOMY/EDGE 用户管理自己发起或参与的项目，处理项目状态与授权进度。

### 主界面元素
- Tab 筛选：我发起的 / 我处理的 / 全部。
- 状态筛选：全部 / 已归档 / 审核中。
- 计算模式筛选。
- 项目卡片：
  - 项目名称、计算模式
  - 参与机构
  - 训练流数、任务数
  - 创建时间
  - 状态标签：已归档、审核中
  - 授权状态标签：当受邀方未全部通过时显示
- 操作：进入项目、编辑（仅发起方且未归档）、P2P 项目按钮组。

### 业务规则
- 仅项目发起方可编辑项目信息。
- 归档项目不可编辑。
- 项目创建后进入 `REVIEWING` 状态，所有受邀机构同意后变为 `APPROVED`。

---

## 三、创建项目

### 页面目标
引导用户完成项目创建，包括基础信息、计算模式、参与节点/机构、训练流模板。

### 主界面元素（分步表单或抽屉）

#### 步骤 1：基础信息
| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 项目名称 | Input | 是 | 全局唯一或节点内唯一 |
| 项目描述 | TextArea | 否 | 项目背景 |
| 计算模式 | Radio | 是 | MPC 管道 / TEE 枢纽 |
| 训练流模板 | Select | 否 | 选择预设模板可快速初始化 DAG |

#### 步骤 2：选择参与节点/机构
- 从已注册节点或合作节点中选择。
- 显示节点类型匹配校验（MPC 项目不能选 TEE 节点等）。

#### 步骤 3：确认创建
- P2P 模式：创建后生成投票请求，跳转到消息中心或项目列表。
- CENTER 模式：直接创建成功，进入项目空间。

### 业务规则
- 项目名称需校验唯一性与长度限制。
- 计算模式一旦选定，项目内组件与菜单随之确定。
- 选择模板后，进入 `/dag` 时画布已预置示例节点。
