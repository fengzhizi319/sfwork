# 页面需求：登录与新手引导

## 一、登录页（/login）

### 页面目标
提供统一的账号密码登录入口，根据用户平台类型与首次登录状态跳转不同首页。

### 主界面元素
- 账号输入框。
- 密码输入框（前端 SHA256 哈希后提交）。
- 登录按钮。
- 错误提示区。

### 登录后跳转规则

| 平台类型 | 是否首次 | 跳转页面 |
|---|---|---|
| AUTONOMY | - | `/edge?ownerId=...` |
| EDGE | - | `/node?ownerId=...` |
| CENTER | 是 | `/guide` |
| CENTER | 否 | `/` |
| CENTER 下的 EDGE 子账号 | - | `/`（跳过引导） |

### 业务规则
- 登录失败超过阈值后账号锁定。
- Token 默认 24 小时，滑动更新。

## 二、新手引导（/guide）

### 页面目标
首次登录的 CENTER 管理员了解平台核心概念，并通过 Tour 逐步熟悉界面。

### 主界面元素
- 概念介绍页：隐私计算、节点、项目、训练流等。
- 步骤引导：
  1. 创建/注册节点。
  2. 创建项目并邀请参与方。
  3. 上传/授权数据。
  4. 编排训练流并运行。
  5. 查看结果与模型。
- 顶部提供“体验新手引导”重新进入入口。

### 业务规则
- 仅 CENTER 平台首次登录进入。
- 引导过程中支持跳过。
- 关键页面（项目列表、数据授权等）使用 Tour 组件定位元素。
