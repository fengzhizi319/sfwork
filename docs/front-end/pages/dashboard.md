# 页面需求：Dashboard（/dashboard）

## 页面目标
为 CENTER 平台管理员提供一个**全局概览页**，快速了解平台规模、最近动态和系统健康。

## 用户故事
- 作为管理员，我希望登录后第一眼看到项目/节点/数据表/训练流的数量，从而掌握平台使用规模。
- 作为管理员，我希望查看最近创建的项目和节点，快速进入详情。
- 作为管理员，我希望看到 CPU/内存/磁盘/网络的健康状态，及时发现系统异常。

## 前置条件
- 用户已登录且 `platformType === CENTER`。
- 后端返回首页统计数据。

## 主界面元素

### 顶部统计卡片
横向排列 4 张卡片：

| 卡片 | 指标 | 交互 |
|---|---|---|
| Projects | 项目总数 | 点击跳转 `/home?tab=project-management` |
| Nodes | 注册节点数 | 点击跳转 `/nodes` |
| Data Tables | 数据表总数 | 点击跳转 `/data-table` |
| Graphs | 训练流总数 | 点击跳转 `/graphs` |

### 最近动态区
- **Recent Projects**：最近 5 个项目，显示名称、创建时间，点击可进入项目。
- **Recent Nodes**：最近 5 个节点，显示名称/地址/状态，点击可进入节点详情。

### 系统健康区
- CPU 使用率
- 内存使用率
- 磁盘使用率
- 网络状态（连通性/带宽提示）

> 当前部分占位值可用，后续需对接后端真实指标接口。

## 操作与反馈

| 操作 | 反馈 |
|---|---|
| 点击统计卡片 | 跳转到对应列表页 |
| 点击最近项目/节点 | 跳转到对应详情/项目空间 |
| 页面加载失败 | 显示错误提示，支持重试 |

## 业务规则
- 仅 CENTER 平台可见。
- 统计数字需要实时或准实时刷新。
- 系统健康指标超过阈值时（如 CPU > 80%）需要高亮告警。
