# 02 角色与权限

## 2.1 平台类型（Platform Type）

SecretPad 前端根据后端返回的 `platformType` 决定整体界面形态。

| 平台类型 | 定位 | 典型入口 |
|---|---|---|
| **CENTER** | 中心管理平台，可注册多个 EDGE 节点，统一项目与数据治理 | `/` 首页：节点注册 + 项目管理 |
| **EDGE** | 边缘节点账号视角，持有实际数据与 Kuscia Domain | `/edge?ownerId=xxx` 工作台 |
| **AUTONOMY** | P2P 自治机构，无中心节点，机构间点对点直连 | `/edge?ownerId=xxx` 或 `/my-node` |

## 2.2 部署模式（Pad Mode）

影响项目创建时可选的计算模式与顶部菜单。

| 模式 | 说明 | 对应前端表现 |
|---|---|---|
| **MPC** | 管道模式，多方安全计算/联邦学习 | 项目支持“训练流”“模型管理”“周期任务” |
| **TEE** | 枢纽模式，可信执行环境 | 项目支持“项目数据”“模型训练”，模型管理按 TEE 规则 |
| **ALL-IN-ONE** | 一体化部署 | 同时暴露 CENTER 与 EDGE 能力 |

## 2.3 账号类型与可见菜单

| 账号 | 登录后跳转 | 可见菜单 | 特殊说明 |
|---|---|---|---|
| CENTER 管理员 | `/` 或 `/guide` | 节点注册、项目管理、消息中心 | 首次登录进入新手引导 |
| CENTER 下的 EDGE 子账号 | `/` | 仅项目管理 | 隐藏节点注册 |
| EDGE 账号 | `/edge` | 工作台、数据源、数据管理、合作节点、我的项目、结果管理 | 必须带 `ownerId` |
| AUTONOMY 账号 | `/edge` 或 `/my-node` | 工作台、我的节点、消息中心、P2P 项目 | 可切换节点 |
| 内置节点账号（alice/bob/tee） | 直接进入 | 对应节点工作台 | CENTER 平台可直接登录 |

## 2.4 权限控制层级

### 路由级权限
通过 Umi `wrappers` 控制页面可访问性。

| Wrapper | 含义 |
|---|---|
| `center-auth` | 仅 CENTER 平台可访问，如 `/dashboard`、`/nodes` |
| `p2p-center-auth` | CENTER 或 AUTONOMY 可访问，如 `/dag`、`/record` |
| `edge-auth` | 仅 EDGE 平台可访问，如 `/node` |
| `basic-node-auth` | URL 必须携带 `ownerId`，如 `/edge`、/message` |
| `p2p-edge-center-auth` | AUTONOMY/EDGE/CENTER 中具备节点视角的账号 |
| `component-wrapper` | 预加载组件解释器与组件树，DAG 相关页面需要 |
| `login-auth` / `login-wrapper` | 登录态校验 |
| `guide-auth` | 新手引导路由权限 |

### 组件级权限
通过 `AccessWrapper` / `hasAccess` 根据 `platformType` 或 `deployMode` 条件渲染按钮/菜单。

示例：
- 仅 MPC 模式显示“模型管理”“周期任务”顶部菜单。
- 仅 AUTONOMY 模式显示 P2P 项目操作按钮。

### 操作级权限

| 规则 | 说明 |
|---|---|
| P2P 项目 | 仅发起方可编辑/删除/发布模型 |
| 训练流 | 非本方创建的训练流仅可查看 |
| 归档项目 | 不可编辑、不可发布模型 |
| 合作节点 | 有运行中任务时不可删除 |
| 数据表 | 已授权到项目的数据表不可删除 |
| 内置节点 | 不可编辑/删除 |
| TEE 下载 | 需通过消息中心审批后才能下载 |

## 2.5 数据权限暗示

虽然前端不直接做数据鉴权，但需通过界面提示让用户理解数据可见范围：

- **项目数据表**：仅在当前项目内可见，授权关系需明确展示。
- **节点数据表**：不同节点下的数据表列表隔离。
- **结果产物**：按来源项目和所属节点展示。
- **消息/审批**：按“我处理的 / 我发起的”区分，未处理消息数显示在顶部铃铛。
