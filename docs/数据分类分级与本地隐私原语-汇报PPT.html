现实痛点：数据“黑箱”与保护脱节

痛点 具体表现 后果
数据“黑箱” 平台不知道接入数据里有没有身份证号、基因序列、HIV 诊断 高敏感数据可能以明文形式参与计算或导出
策略与计算脱节 安全策略写在文档里，实际执行靠人工判断 误用、漏用保护算法，合规风险高
使用门槛高 DP 的 ε、K-匿名的 K、脱敏规则对业务开发不透明 业务系统不敢用、不会用、乱用
接入方式单一 只有批量 DAG 组件，没有本地实时能力 无法覆盖 API 脱敏、医生工作站、大模型网关
参数无法治理 参数散落在代码、配置文件、UI 面板中 无法审计、无法版本化、无法自动推荐

数据分类分级：给数据贴“安全标签”

五级敏感标准

L5 极高风险 基因序列、罕见病原始样本 → 强制入 TEE
L4 高风险 HIV / 精神病 / 癌症完整病历 → MPC / FL
L3 中风险 身份证号、手机号、诊疗记录 → DP / K-匿名 / 脱敏
L2 低风险 去标识化统计级数据 → 联盟内受限共享
L1 公开级 医院公开运营数据 → 公开发布

三层漏斗式检测引擎

第一层 · Rule Engine

RE2 正则 + FST 字典，微秒级，100% 召回目标

第二层 · Small-NER

ONNX 轻量模型，毫秒级，医疗命名实体识别

第三层 · LLM Classifier

医疗垂类大模型，秒级，旁路异步复核

07 / 12

L1~L5 协同路由策略

数据等级 协同计算策略 本地保护动作
L5 原则上禁止进入 FL/MPC 明文计算；必须入 TEE 或硬件隔离 禁止出域、端到端加密
L4 进入 MPC / VFL，数据可用不可见；输出侧加 DP DP（ε≤0.1）、严格审计
L3 可进入 FL / MPC，但需先本地脱敏或 K-匿名 K-匿名（k≥5）、脱敏、DP（ε≤1.0）
L2 可在联盟内 FL / MPC 共享 轻量脱敏、受限查询
L1 无需 FL/MPC，直接公开 无

端到端示例：医院 A 表经分类分级识别 L5 字段 → L5 剥离/入 TEE，L4 进入 MPC 统计，L3 先脱敏 → PSI 对齐共同患者 → MPC 聚合 → 输出侧 DP 加噪 + 脱敏 → L2 结果出域。

总结

本项目不是单纯增加几个算法，而是构建一套
“识别 → 决策 → 执行 → 审计” 的隐私保护闭环。

分类分级让平台“看得见”

自动识别 L1~L5 敏感等级，输出可消费的 SecurityTag。

本地隐私原语让平台“管得住”

脱敏、K-匿名、DP、QOL 覆盖单点与批量场景。

FL / MPC 让协同计算“不可见”

跨机构联合建模与统计，原始数据始终不出域。

参数治理与审计让风险“可追溯”

模板、Profile、预算台账、审计日志四位一体。
