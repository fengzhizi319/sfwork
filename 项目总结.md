# SFWork 项目总结

本文档详细介绍了 sfwork 工作空间内的各个项目，包括其编程语言、架构设计和技术细节。

## 项目概览

### 1. Kuscia
**项目描述**: Kuscia 是 SecretFlow 生态系统的底层编排引擎，作为运行时基础设施管理联邦学习场景中的分布式计算任务、数据资产和多方通信。它使用 Kubernetes CRD（自定义资源定义）来管理作业、任务和数据资产。

**主要语言**: Go

**关键组件**:
- **Agent**: 负责与 Kubernetes API 服务器通信
- **Controllers**: 管理 KusciaJob、KusciaTask、DomainData 等 CRD
- **DataMesh**: 提供数据访问和授权层
- **Gateway**: 管理跨域通信
- **KusciaAPI**: 为外部系统提供 gRPC/REST API
- **Scheduler**: 负责任务调度和资源分配
- **CoreDNS**: 为集群内部服务发现提供 DNS 解析
- **Utils**: 通用工具包，包含加密、网络、文件操作等

**常用编程技术**:
- **Kubernetes 控制器模式**: 使用 Informers 和 WorkQueue 监听资源变更并调和期望状态
- **依赖注入**: 使用构造函数和接口实现松耦合设计
- **错误处理**: 使用 pkg/errors 进行详细的错误包装和堆栈追踪
- **结构化日志**: 使用 Zap 日志库进行高性能日志记录
- **配置管理**: 使用 Viper 进行配置文件处理
- **gRPC 服务**: 实现组件间的高效通信
- **CRD 管理**: 定义和管理自定义 Kubernetes 资源
- **安全机制**: 实现 mTLS 认证和授权
- **测试策略**: 广泛使用表格驱动测试和 gomock 进行模拟测试
- **并发控制**: 使用 Go 协程和通道进行并发处理

**核心技术特性**:
- **分布式协调**: 使用 etcd 作为分布式键值存储
- **任务调度**: 支持复杂的任务依赖关系和资源调度
- **多租户隔离**: 为不同域提供数据和计算隔离
- **网络管理**: 实现安全的跨域数据传输
- **监控告警**: 提供丰富的指标和健康检查机制

### 2. SecretFlow
**项目描述**: SecretFlow 是一个统一的隐私保护计算框架，支持安全多方计算（MPC）、同态加密（HE）和可信执行环境（TEE），在不暴露原始数据的情况下执行联合分析和建模。

**主要语言**: Python

**关键组件**:
- **Component**: 隐私保护算法的构建块
- **Compute**: 分布式计算基础设施
- **Data**: 数据处理和预处理模块
- **Device**: 不同计算设备的抽象层（SPU、HEU 等）
- **ML**: 适应隐私保护的机器学习算法
- **Preprocessing**: 数据预处理和特征工程
- **Stats**: 统计分析功能
- **Security**: 安全协议和加密算法实现
- **Kuscia 集成**: 与 Kuscia 协调的适配器

**常用编程技术**:
- **函数式编程**: 大量使用函数式模式进行数据变换
- **类型提示**: 广泛使用 Python 类型注解提高代码可维护性
- **上下文管理器**: 使用 'with' 语句进行资源管理
- **装饰器**: 自定义装饰器用于日志记录、验证和性能监控
- **协议缓冲区**: 用于序列化和进程间通信
- **JAX**: 用于机器学习运算的数值计算库
- **依赖注入**: 配置和服务管理
- **异步/等待**: 异步编程用于 I/O 操作
- **测试策略**: 使用 pytest 进行单元测试，包含 fixture 和参数化测试

**核心技术特性**:
- **多方计算**: 实现高效的 MPC 协议
- **安全聚合**: 保护中间计算结果
- **差分隐私**: 添加噪声保护数据隐私
- **联邦学习**: 支持分布式模型训练
- **设备抽象**: 统一的硬件接口抽象
- **性能优化**: 针对大规模数据集的优化算法

### 3. SecretPad
**项目描述**: SecretPad 是 SecretFlow 生态系统的 Web 管理控制台，提供用户友好的界面进行数据管理、联邦学习任务创建和结果可视化。它处理业务逻辑、持久化和与 Kuscia 的通信。

**主要语言**: Java

**关键技术栈**:
- **Spring Boot**: 微服务框架，自动配置
- **Spring MVC**: Web 层框架
- **MyBatis**: SQL 映射和数据库交互
- **Spring Security**: 安全框架
- **MySQL/SQLite**: 数据持久化
- **Maven**: 项目构建工具

**关键组件**:
- **API 层**: 与前端通信的 RESTful API
- **服务层**: 业务逻辑和工作流编排
- **持久层**: 数据库交互和实体管理
- **管理器层**: 领域特定操作和验证
- **Web 层**: 请求处理的控制器层
- **公共组件**: 共享工具类、枚举和常量

**常用编程技术**:
- **Spring 框架**: 依赖注入、事务管理和 MVC 架构
- **MyBatis**: SQL 映射和数据库交互
- **Spring Boot**: 自动配置和嵌入式服务器功能
- **MVC 模式**: 模型、视图和控制器之间的清晰关注点分离
- **异常处理**: 自定义异常层次结构和全局异常处理器
- **验证**: 使用 JSR-303 注解进行 Bean 验证
- **配置属性**: 类型安全的配置属性
- **事务管理**: 使用 @Transactional 的声明式事务处理
- **安全机制**: 认证和授权机制
- **测试策略**: 使用 JUnit 和 Mockito 进行单元和集成测试

**核心技术特性**:
- **双存储架构**: SecretPad DB 支持历史追溯和业务查询，Kuscia 实现分布式协调和弹性扩展
- **数据一致性**: 通过 Spring @Transactional 保证数据一致性
- **故障补偿机制**: 确保系统的可靠性和容错性
- **项目管理**: 任务执行记录、状态跟踪和数据持久化
- **权限管理**: 细粒度的访问控制和数据授权

### 4. SecretPad 前端
**项目描述**: SecretPad 的前端组件，使用现代 Web 技术构建。提供直观的用户界面用于管理隐私保护计算、可视化数据流和监控作业进度。

**主要语言**: TypeScript, JavaScript

**关键技术栈**:
- **React**: 基于组件的 UI 架构
- **Umi**: 可插拔的企业级 React 应用框架
- **Ant Design**: 企业级 UI 设计系统
- **ESLint/Stylelint**: 代码质量和风格强制
- **Husky/Lint-staged**: Git hooks 用于代码质量检查
- **Nx**: 单体仓库管理
- **Dva**: 基于 Redux 的轻量级数据流方案
- **Less**: CSS 预处理器

**关键组件**:
- **数据表管理**: 数据表的创建、编辑和查看功能
- **项目管理**: 联邦学习项目的创建和管理
- **图编辑器**: DAG 任务流程的可视化编辑
- **结果展示**: 计算结果的可视化呈现
- **节点管理**: 参与方节点的管理
- **权限管理**: 数据访问权限的配置

**常用编程技术**:
- **组件化架构**: 可重用和模块化的 UI 组件
- **Hooks**: 使用 React Hooks 进行状态管理和副作用处理
- **TypeScript**: 静态类型检查以提高代码可靠性
- **状态管理**: Context API 和可能的 Redux/Zustand 用于全局状态
- **异步操作**: Promise/async-await 用于 API 调用
- **模块打包**: ES 模块和摇树优化
- **测试策略**: Jest 和 React Testing Library 进行单元和集成测试
- **表单处理**: 表单库用于数据收集和验证
- **国际化**: 多语言支持

**核心技术特性**:
- **响应式设计**: 适配不同屏幕尺寸
- **可视化编辑**: 拖拽式任务流程编辑器
- **实时协作**: 多用户协同工作的支持
- **性能优化**: 虚拟滚动、懒加载等优化技术
- **用户体验**: 流畅的交互和直观的操作流程

### 5. DataMesh
**项目描述**: DataMesh 是 Kuscia 生态系统中的数据访问和授权层。它抽象底层数据源并为联邦学习场景提供安全的访问控制。它管理 DomainData、DomainDataSource 和 DomainDataGrant 资源。

**主要语言**: Go (作为 Kuscia 项目的一部分)

**关键组件**:
- **MetaServer**: 集中式元数据管理
- **DomainData 服务**: 数据资产管理
- **DomainDataSource 服务**: 数据源注册和连接
- **Operator**: CRD 的 Kubernetes 操作器
- **DataProxy**: 数据代理，提供安全的数据访问接口

**常用编程技术**:
- **微服务架构**: 面向服务的设计原则
- **API 优先设计**: 使用 Protocol Buffers 的明确定义契约
- **认证与授权**: 基于角色的访问控制和细粒度访问控制
- **数据验证**: 输入清理和模式验证
- **事件驱动架构**: 响应对 Kubernetes 中资源变化
- **连接池**: 高效的数据源资源管理
- **缓存**: 频繁访问元数据的性能优化
- **监控**: 指标收集和健康检查

**核心技术特性**:
- **数据虚拟化**: 抽象物理数据源，提供统一访问接口
- **安全访问控制**: 细粒度的数据访问权限管理
- **元数据管理**: 集中管理数据资产的元信息
- **跨域数据共享**: 安全的跨组织数据共享机制
- **数据血缘**: 追踪数据的来源和转换过程

## 架构集成

这些项目按以下方式协同工作：

1. **前端 (TypeScript/React)** 与 **SecretPad 后端 (Java)** 通信
2. **SecretPad 后端 (Java)** 通过 API 与 **Kuscia (Go)** 编排
3. **Kuscia (Go)** 管理执行环境并与 **SecretFlow (Python)** 协调
4. **SecretFlow (Python)** 执行实际的隐私保护计算
5. **DataMesh (Go)** 提供跨域的安全数据访问

## 跨项目通用模式

- **gRPC 通信**: 标准化的服务间通信
- **Protocol Buffers**: 跨语言的一致数据序列化
- **容器化**: 基于 Docker 的部署
- **Kubernetes 编排**: 部署和扩展平台
- **可观测性**: 跨整个技术栈的日志、指标和追踪
- **安全性**: 端到端加密和认证机制
- **CI/CD**: 自动化测试和部署流水线
- **文档**: 全面的 API 和架构文档
- **数据持久化**: 支持历史追溯和业务查询的持久化存储
- **故障恢复**: 包括补偿机制的容错设计
